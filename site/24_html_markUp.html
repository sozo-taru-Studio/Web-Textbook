W3C（ダブリュ３シー）について - HTML

category: HTML

World Wide Web Consortium (W3C)

W3Cの仕様書等の文書の日本語訳集

ワールドワイドウェブソローシアムは インターネットとそのリソースの規格や標準化を勧告する団体です。デベロッパーにとって身近な部分では HTMLで使われる要素の役割やCSSのプロパティやバリューを草案から設計そして勧告したりする組織です。インターネットで情報がみれるのもこの団体のおかげです。

W3Cミッション の役割

W3Cミッション

各社が開発するインターネットブラウザ機能拡充、互換性や解釈の誤解をうまないようにしています。 そのためユーザーやインターネット開発業者の不利益が防げるやくわりがあります。

W3Cはすべてひとにインターネットにアクセスできるよう活動しています。 アクセシビリティを重要視し、より多くの人で様々なデバイスでインターネットで情報がやり取りできるように活動しています

w3c,ブラウザメーカー,デベロッパーの役割

W3CはHTMLCSSの機能、記述方法などを設計標準化する。

ブラウザメーカーはW3Cの勧告に沿った機能をブラウザに実装する。（努力をしている）

制作者はW3Cの勧告あったにコーディングをしユーザーにwebsiteを公開する。

HTMLやCSSの仕様を正確に知るには

HTML&CSSのリファレンスや仕様を確認する場合ここで確認するのが正確です。 またブラウザメーカ-のリファレンスなどもみなW3Cの勧告をもとに辞書などを作成しています。 講師に教わることの元はここにのっているとおもってください。

世界中のwebサイトはW3Cに従順なのか？

W3Cは年単位で勧告されたものを含め言っている事が微妙に変わります。 特に新しいく実装されたりまだ協議中のタグなどは使い方を含め変更があります。 ブラウザ側も後方互換性を考えると廃止予定のタグも表示しなくてはならなかったりと非常に判断がつきづらいです。

結局安定してつかわれてきたタグや技術が主流だったりしますし、w3cがやブラウザメーカーが目指してきた理念や理屈なんかを理解しながらマークアップしてもらうとあまり大きな間違いのあるコーディングは避けれるれるのではないでしょうか？

ちょっと宗教じみていますが、w3cの理念を理解して意味をおもんばかれる努力をしなければいつまでたっても独りよがりで独善的なマークアップになります。
