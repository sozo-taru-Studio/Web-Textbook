  • CSS　閲覧者に伝わりやすく装飾

通常のUA(ブラウザ)の挙動をプログラムで変更して、よりユーザーに便利に働くようにするためにブラウザの処理を変更するために書きます。

コンピュータープログラミングの簡単な概要

プログラミング言語とはコンピューターに処理をさせるための言語 コンピューターとの手続き

スクリプト(script)とは「台本、脚本」という英語の意味で簡易なアプリケーションの操作や設定を書けるプログラム。
scriptは手軽で簡単である程度融通が効き学習のハードルの低いと感じさせます。

javascriptの概要

  • javascriptはインタプリタ型 (解読しながら実行する)プログラミング言語
  • javascriptはクライアントサイド（ブラウザ）で実行されます。
    クライアントサイドjavascriptを開設しています。
  • サーバサイド JavaScrip・・・・Node.jsなど、サーバーサイトプログラムとしてjavascriptが最近（2014年ごろ）ではつかわれはじめています。

各ブラウザにはjavascript言語を解読（インタプリタ）して実行をするエンジンがついています。

javascript言語はwebページがどうのように立ち振る舞うかを設定するプログラミング言語です。

プログラムの処理の種類

インタプリタ型とは別に処理方式の違うコンパイル型のプログラミングがあります。

コンパイル型：プログラム言語をアプリケーションが理解できるように解読する行程が必要なプログラムです。 行程は「プログラミング コンパイル 実行」のプロセスが必要です。アプリケーションの作成など大規模なプログラムに向いています

標準化 - javascript もHTMLやCSSとなどと同じく世界標準化されています。

ブラウザによってプログラミングの処理が微妙に違うとプログラマーは困ります。標準化作業をしているチームの名前が「ECMA」というチームです。でそこから ECMAScriptと呼ばれるようになりました。

ECMAScript 仕様

JavaScript は国際的なプログラミング言語を提供できるように Ecma International によって標準化されています (ECMA は、以前は European Computer Manufacturers Association の頭文字をとったものでした)。この標準化された JavaScript のバージョンは ECMAScript と呼ばれ、この標準をサポートするアプリケーションはすべて同じ動作になります。企業は、JavaScript の実装開発にこのオープンな標準言語を利用できます。ECMAScript 標準は、ECMA-262 仕様に文書化されています。JavaScript バージョンと ECMAScript 標準化の各版との違いについて知りたい場合は JavaScript の新機能をご覧ください。

「Javascript」と「Java」の違い

詳しく知りたければMDNのリンクをよんでみてください。
インドとインドネシアぐらい違う。目的も使うところも違う言語です。 ですから習い始めの人や入門落ちした人がよく「ジャバ」って呼んでらっしゃるのを聞きますが、そうゆう呼び方は正確につたわらないし知らなさ加減がしれますのでジャバスクリプトやジェイエス（JS）などと呼びJavaプログラムと聞き分けられるようにしてください。
