関数を定義する

関数 - JavaScript | MDN

関数はJavaScriptの基本的な構成要素のひとつです。また関数は、JavaScript の手続き ― つまり、タスクや値計算を実行する文の集まりです。関数を使うには、呼び出したいスコープ内のどこかでそれを定義する必要があります。

関数の目的

プログラムは簡単な処理しかできません。
そこで必要な機能を得る為に複雑な処理をあらかじめまとめて書いておきます。まとめた処理である関数を呼び出して実行します。
関数は同じ処理を何度も書いたりしなくて済みます。関数を使うことでプログラミングのメンテナンスの軽減やプログラミングの記述量を減らすことができます。

関数を例えると

関数とは人間の行動で例えると習慣や文化です。寝るや起きる食べるなどは生き物に備わった処理です。 毎朝7時に会社に出社する機能は自分の中に組み立てられた処理の蓄積のもとに達成できる習慣になるわけです。
皆さんも会社や学校に至るまでにやっている処理や習慣はいくつあるでしょうか？例えば歯を磨く、朝食を食べる、お弁当作る、などなど。やらなければいけないことが頭のなかですぐにリストアップして思い出せるはずです。 プログラムも簡単な処理をまとめておいてすぐに実行できる機能があります。この機能を関数と読んでいます。

関数の宣言

関数の定義（関数の宣言や関数定義文 (function statement) とも呼ばれます）は「function」 キーワードとそれに続く以下の内容で構成されます。

宣言の仕方

  1. 関数の名前。: functionの後に半角スペースで区切り命名する。function functionName
  2. 関数への引数のリスト。: 丸括弧でくくり、コンマで区切ります。(arg0,arg1)
  3. 関数を定義するJavaScriptの文。波括弧 {} でくくります。{//まとめる処理};

構文 - 関数宣言(function statement)

構文の注意

function 半角スペース 関数名 の順番で宣言してください。半角スペースがなければ定義として認識されません。

即時関数の目的はトップレベルで記述する処理を関数化することで変数スコープを限定し、変数の汚染を防ぐ。

言い換えれば、極力グローバル変数を減らしメモリーを解放したり上書きを回避するテクニック。

即時関数のメリットと主な用途

  1. 即時関数は本質的には通常の名前付き関数と同じ。つまり「即時関数を使わなければ実現できない処理」といったものはあり得ない

引数 - (argument)

argumentは議論や議論のテーマという意味です。

プログラミングでの引数（ひきすう）は関数の（）カッコ内にわたされます。
引数はキーワードとして関数の中で処理される値です。

関数定義時小かっこでわたされるキーワードを仮引数とよびます。仮引数を複数て設定したい場合は「,」カンマで区切ります。

引数と変数は似てますが違います。javascriptでは引数には var などの宣言文は必要ありません。むしろ構文エラーをはきます。

関数内の暗黙的変数との区別は仮変数のキーワードです。

引数は関数のなかの回路のような情報の流れ道です。
引数を設定することでで柔軟な関数の設計ができます。

先ほどの「朝7時に出社する」習慣の比喩で例えるなら引数は朝7時の部分です。
「朝8時」と引数で渡すことで、関数の処理は朝8時の処理をします。引数に渡した条件で処理を変化させて処理に柔軟性をもたせることもできます。

関数の引数がない場合undifinedが返されます。
