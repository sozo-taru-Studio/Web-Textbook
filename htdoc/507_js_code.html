ブロック文

たいていの場合、ブロック文は処理のグループ化に使います。ブロックは、1組の波括弧「{}」で区切られます。 たいていの場合とは関数の処理内容や分岐後の処理などにつかわれます。

CSSと同じで開始の波カッコ「{」は閉じる波カッコ「}」で閉じられる。ネスト構造(階層構造)を意識してプログラミングしていただくとプログラミングミスが減ります。

重要 : ECMAScript 2015 より前の JavaScript にはブロックスコープがありません。ブロック内で導入された変数のスコープは、そのブロックがある関数やスクリプトになります。またそれらの変数を設定した影響は、そのブロックを越えて持続します。つまり、ブロック文はスコープを定義しないということです。「独立した」ブロックも正しい構文ですが、CやJavaのブロックで提供されるものとは異なる結果をもたらします。例えば :

var x = 1;
{
  var x = 2;
}
console.log(x); // 2 を出力

他のプログラミングと違ってブロック文のなかの変数は外の変数と同じ扱いになり書き換えられます。（変数のスコープ）

条件文

条件文は、指定した条件が true ならば実行される命令の集まりです。JavaScript は if...else と switch の 2 つの条件文をサポートしています

if文は自由度が高いですが構文が複雑になりがちです。
またお約束が少なく覚えやすいのが利点です。

if...else 文

論理条件が true の場合に、文を実行するには if 文を使用してください。オプションの else 節を使用すると、条件が false の場合にも文を実行することができます。if 文は次のように使用します :

conditionは条件文です。比較演算や理論演算をいれて真偽値をいれます。

condition文が真の場合処理し偽の場合ブロック内の処理はしません。


// trueの処理する。

  var Obj;//変数宣言
  Obj = {};//object定義

  if (typeof Obj === 'object') { //変数Objの型番が`Object`
  /*trueのときの処理*/
   var varType = typeof Obj;
    console.log(varType);
  }
  
  
// trueの処理をしません。
var obj2;
if (typeof obj2 === 'object') { //変数obj2の型番が`Object`
/*trueのときの処理*/
  var varType = typeof obj2;
  console.log(varType);
}

変数obj2が未定義のため処理しません。

条件に合ったとき処理と合わなかったときの処理がある場合

condition文が真の場合処理し偽の場合、次のelseの処理をする。

次のように else if を使用して文を重ねることで、複数の条件を順々にテストすることができます

「false」と評価される値

以下の値はfalseに評価されます。

条件文内に入れる評価値をしることで条件分岐の幅が増えます。

  • false:真偽値の値を返り値にする関数
  • undefined:変数内が未定義
  • null：DBへのクエリが空
  • 0：数値型でゼロ
  • NaN：数値かどうか確認して数値ではなかった。not a Number
  • 空の文字列 ("")
